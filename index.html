Progetto di Reti di Calcolatori

luca.mori@studenti.unimi.it - AA 21/22

Il progetto si pone come obiettivo di applicare i concetti e le tecniche di implementazione dei servizi e protocolli discussi ed esaminati durante il corso, e consiste nella progettazione di una rete universitaria con l'utilizzo del software IMUNES usato nelle ore di laboratorio.

Tematiche trattate: DHCP, DNS, FTP, HTTP, IPTABLES, SMTP, Trunking, VLAN
Dipendenze (in aggiunta alla macchina virtuale LabReti2019): ifupdown

Si consideri una sede universitaria composta da:

  • 250 postazioni riservate agli uffici amministrativi
  • 500 postazioni riservate agli uffici professori
  • 4000 postazioni riservate agli studenti
  • 2 laboratori per studenti (250 postazioni/laboratorio)
  • 5 laboratori di ricerca (250 postazioni/laboratorio)

Tutte le postazioni sono concentrate in tre edifici:

In ogni edificio sono presenti switch collegati in cascata, ognuno dei quali gestisce le postazioni di un singolo piano. Gli switch in cascata sono quindi collegati ad uno switch centrale collegato al server DHCP per l'assegnamento dinamico degli indirizzi. Infine, lo switch centrale è collegato ad un router che regola la comunicazione tra la rete locale e i server (DNS, Web, SMTP, FTP) attraverso regole iptables.

Per la realizzazione della rete si è scelto l'approccio di tipo Software-defined networking: per mettere in contatto le interfacce appartenenti alla stessa VLAN tra diversi piani ed edifici, vengono settate porte VLAN Trunking (802.1Q) tramite Open vSwitch.

L'assegnazione degli IP è dinamica, e viene effettuata tramite il server DHCP, che espone un'interfaccia virtuale per ogni VLAN sull'interfaccia fisica eth0, tramite il deamon NetworkManager. La medesima soluzione viene applicata al router sull'interfaccia fisica eth0. A seconda del tag VLAN presente nel frame ethernet questo verrà gestistito dall'interfaccia virtuale corrispondente a quella rete.

Oltre al server DHCP, nella rete universitaria sono presenti i seguenti server, il cui accesso è controllato tramite regole iptables presenti sul router:
  • DNS Server ns.university.edu
    Traduce gli indirizzi IP degli altri server, accessibile da tutta la rete interna
  • Web Server www.university.edu
    Gestisce i servizi web sulla porta 80, accessibile da tutta la rete interna
  • SMTP Server smtp.university.edu
    Gestisce la posta elettronica dell'ateneo, accessibile solo dalla rete studenti, professori e amministrazione
  • FTP Server ftp.university.edu
    Gestisce la posta elettronica dell'ateneo, sia in modalità attiva che passiva, accessibile solo dalla rete professori e amministrazione
I pacchetti ICMP transitano liberamente. Il computer avente MAC Address 42:00:aa:00:00:30 ed appartenente alla VLAN Administration può connettersi ai vari server tramite SSH.
